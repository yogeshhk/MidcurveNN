# Phase III: Text-Based Midcurve Prediction

## Overview

This folder implements the **text/LLM-based approach** to midcurve prediction. Shape geometry is
expressed as a **B-rep JSON text sequence** and fed to a sequence model (LLM fine-tuned with LoRA)
or used in few-shot prompt-based generation.

Integrated from [MidcurveLLM](https://github.com/yogeshkulkarni/MidcurveLLM).

## Data Format

### BRep JSON (canonical — `data/brep/`)

```json
{
    "ShapeName": "T",
    "Profile":   [[x, y], ...],
    "Midcurve":  [[x, y], ...],
    "Profile_brep": {
        "Points":   [[x, y], ...],
        "Lines":    [[i, j], ...],
        "Segments": [[line_idx, ...], ...]
    },
    "Midcurve_brep": { "Points": [...], "Lines": [...], "Segments": [...] }
}
```

Key invariant: `Lines` and `Segments` (topology) are **never changed** by transformations —
only `Points` coordinates are updated.

### CSV columns (`data/csvs/`)

| Column | Content |
|---|---|
| `ShapeName` | e.g. `T_rotated_45` |
| `Profile` | JSON string — flat `[[x,y],...]` |
| `Midcurve` | JSON string — flat `[[x,y],...]` |
| `Profile_brep` | JSON string — full BRep sub-dict |
| `Midcurve_brep` | JSON string — full BRep sub-dict |

## Common Commands

### Regenerate CSV datasets from base shapes

```bash
cd src/text_based/utils
python create_brep_csvs.py
```

### Onboard a new shape (.dat + Yogesh txt -> JSON)

```bash
cd src/text_based/utils
python create_brep_jsons.py --shape <ShapeName>
python create_brep_jsons.py --dry-run    # preview only
```

### Fine-tune an LLM (QLoRA)

```bash
cd src/text_based/finetuning
python data_validator.py          # validate CSV data first
python train.py                   # train Qwen/Gemma/Mistral with LoRA
python inference.py --single      # test one prediction
python evaluate.py --model_path Midcurve-Qwen-Coder-v1 --visualize
```

### Run full fine-tuning pipeline

```bash
cd src/text_based/finetuning
python run_pipeline.py --full
```

### Serve predictions via API

```bash
cd src/text_based/finetuning
python model_server.py --host 0.0.0.0 --port 8000
# POST http://localhost:8000/predict
# GET  http://localhost:8000/docs  (Swagger UI)
```

### Run tests

```bash
cd src
python -m pytest text_based/testing/test_text_based.py -v
```

## Key Challenge

Branched midcurves (T, Plus shapes) **cannot be serialized as a simple linear sequence** — the
skeleton has junctions requiring a graph. The BRep format with `Lines` and `Segments` solves this
explicitly. Fine-tuned LLMs must learn to output valid BRep JSON with correct junction connectivity.

See `src/geometry_based/` for the graph-based approach (current best).
