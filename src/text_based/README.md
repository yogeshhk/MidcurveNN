# Phase III: Text-Based Midcurve Prediction

## Overview

This folder contains (or will contain) the **text/LLM-based approach** to midcurve prediction. Instead of rasterizing polygons to images or converting them to graphs, the shape geometry is expressed as a **B-rep JSON text sequence** and fed to a sequence model (transformer / LLM).

## Data

`data/sequences.json` — Generated by `src/utils/prepare_data.py`. Each entry is a dict with:
- `ShapeName`: shape identifier (e.g. "I", "L", "Plus")
- `Profile`: list of `[x, y]` coordinate pairs for the closed polygon
- `Midcurve`: list of `[x, y]` coordinate pairs for the midcurve
- Augmented variants: `Scaled_Profile_*`, `Rotated_Profile_*`, `Translated_Profile_*`, etc.

Regenerate with:
```bash
cd src
python utils/prepare_data.py   # writes text_based/data/sequences.json
```

## Approach (Planned)

The key challenge: **branched midcurves cannot be serialized as a simple sequence** — a Plus or T shape cannot be traced without lifting a pencil. Planned options:

1. **Token-level representation** — Encode profile and midcurve as flat token sequences with special branch tokens. Use a seq2seq transformer (T5 / BART style).
2. **Hugging Face transformers** — Fine-tune a pretrained text encoder-decoder on B-rep JSON input/output pairs.
3. **Graph-of-thought prompting** — Use an LLM to reason about the geometry step by step.

## Status

Phase III is **planned**. The `data/sequences.json` dataset is ready. Model code will be added here as the phase matures.
